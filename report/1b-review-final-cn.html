給 MKT 的結果總結與下一步

資料窗期：2026-06-12 ～ 2026-07-20（GMT+8，39 天，賽事全數結束） 範圍：OT（12bet）平台 資料來源：Admin 後台（核心加總數字）＋ Looker（多維度細節，全 39 天新鮮匯出）

先講結論

  1. 1B 的定義已確認為「賽事期間全平台絕對量」，最終達成率 35.1%。39 天 Admin 口徑 Valid TO 收在 $351.0M（未剔除作廢 $361.9M，36.2%）。差距有多大？達成 1B 要日均 $25.6M，實際日均 $9.0M，等於要求平台在 39 天做出平常 111 天的量。這不是行銷加把勁能補的量級。事件隔離對照給出了實際的增量：世界盃讓全平台較平時基期多做 +3.1%（Sportsbook +14.8%，Non-Sports −7.9%），而 1B 要求的是 +194%，目標與事件實際拉力差了約 60 倍。35.1% 的達成率反映的是目標量級，不是執行力。
  2. 世界盃活動在追蹤系統裡幾乎是隱形的。用「Event」查、用「Source」查，明確標記為 World Cup 的下注量都卡在 0.03%～0.05%，99.80% 的量掛在既有的「Old Event」機制上。一場全平台都在推的世界盃，歸因量低到這種程度，第一個該懷疑的是標籤系統沒接住流量，而不是活動沒人玩。追蹤問題排除之前，這份報告裡所有活動成效數字都只能當下限讀，不是定論。
  3. 在「可被追蹤到」的範圍內，MKT 主控通路的歸因貢獻約 2.3%。Paid Ads、SEO、Social Media、KOL、各幣別 CM 團隊加上其餘小項，歸因加總約 $8.13M，占全平台 Valid TO 的 2.26%；Organic（58.56%）加 Affiliate team（39.18%）吃掉 97.7% 以上。這個 2.26% 跟上一條出自同一套歸因系統，標籤覆蓋有缺陷的話，它同樣被低估。
  4. 供應商/遊戲層級有 12 筆負 GGR，合計虧損約 $168,680（占總 GGR 1.39%），其中 Pronet（PG Sports）與 TCG SEA Lottery 兩項就占了 78%。金額對整體影響不大，但值得 Risk 團隊抽查（詳見風險揭露段落）。

發現一：世界盃把 Sportsbook 拉高 14.8%，但全平台總量只多 3.1%，因為 Non-Sports 同期掉了 7.9%。Sportsbook 占比從基準期的 48.7% 升到 54.2%。也就是說，賽事期間有一部分的量是從其他產品線移過來的，不全是新錢。至於這是「玩家把預算從 Casino/Lottery 挪去投世界盃」的分流，還是 Non-Sports 剛好自然回落，以現有的彙總資料分不出來，要看玩家層級的交叉數據（同一批人是否減少非體育投注）才能確認。

發現二：TO 多了 3.1%，GGR 卻持平。賽事期間 Margin 從 3.48% 滑到 3.37%。方向上可以說世界盃帶來的是「量」不是「利」，但兩邊 GGR 的差距（−0.02%）遠小於方法論本身的誤差（見文末但書），持平是唯一站得住的講法，不能說賺得更少。

發現三：拿基準期對照 1B，目標與事件實際拉力差了約 60 倍。1B 相當於要求賽事期間做出比平時基期多 194% 的量，世界盃實際帶動的全平台增量是 +3.1%。35.1% 的達成率不是執行不力的結果，是目標本身超出了這類事件能拉動的量級。下次訂目標，這兩個數字（基期 $340M、事件增量約 3%）就是起點。

對 MKT 的意義：全平台 +3.1% 看起來不多，但 Sportsbook +14.8% 是實打實的拉抬，Bet User 也多了 3.1%（約 800 人）。下次的重點不是「讓賽事拉更多量」，是兩件事：讓 Sportsbook 的增量少被 Non-Sports 的下滑吃掉（如果分流屬實，交叉銷售的設計就是關鍵），以及把增量做成能歸因的形式，讓 MKT 的貢獻看得見。

給 MKT：這次的證據，跟下次可以怎麼做

證據一：世界盃活動的追蹤幾乎是空的，但這不等於「活動沒用」

Event/Source 兩個獨立維度都測到 0.03%～0.05%，Old Event 吃掉 99.80%，數字涵蓋完整 39 天賽期。

這件事有三種可能，而且不互斥。一，曝光沒進到玩家的下注路徑：世界盃入口可能只擺在活動頁，沒放進首頁或下注流程。二，標籤沒貼對：玩家其實是衝著世界盃來的，但系統沒認出來。Old Event 吃掉 99.8% 流量這件事，讓這個可能性最值得優先排查。三，曝光到了但誘因不夠，玩家還是選了既有機制。哪個才是真正原因，要抽真實下注紀錄才知道，彙總數字回答不了「為什麼」。

證據二：歸因系統看得見的 MKT 主控通路貢獻是 2.26%，但這個數字目前只能當下限讀

Paid Ads＋SEO＋Social＋KOL＋CM團隊＋其餘小項合計約2.26%，Organic＋Affiliate team吃掉97.7%以上。

單看平台側數據，分不出「MKT 真的沒加碼投放」跟「投了但沒被正確歸因」。還有一個結構性的可能：MKT 的投放確實帶來了玩家，但玩家進站後被記成 Organic 或 Affiliate，歸因鏈斷在 landing 之後。真是這樣的話，MKT 的貢獻就藏在那 58.56% 的 Organic 裡，平台側數據永遠看不見。要分辨，只能拿 MKT 的實際投放紀錄（花了多少、投在哪、什麼時候上線）跟這份歸因結果逐段對照。

證據三：39天裡只有一天Margin站上5.7%，隔天決賽反而掉下來

07-19（3強戰/季軍賽當天）是全期Margin最高點，07-20決賽當天卻回落到2.24%。

這是全平台混合 Margin，不是世界盃盤口本身的表現。目前只看得到「發生了什麼」，看不到「為什麼」。可能是決賽當天玩家集中下注在低 Margin 的市場（例如獨贏盤），也可能是其他產品線當天拉低了混合值。兩種都只是假設，要拆到產品線層級的每日數據才能確認。

下次可以怎麼做

先查清楚追蹤到底有沒有壞，再判斷活動有沒有用。抽幾筆確定從 WC banner 進站、或領過 WC 專屬紅利的玩家，看他們實際下注紀錄裡 Event/Source 標籤有沒有貼對。這件事建議排在所有討論之前。追蹤真的有漏洞的話，MKT 的貢獻被嚴重低估，後面所有數字的前提都不成立。

把實際投放金額跟歸因結果對一遍。付費團隊貢獻偏低這件事，需要拿財務真實的媒體投放紀錄比對Looker歸因結果，才能確認是投放不夠、還是投放了但沒被追蹤到。

下次提早佈署，位置也要更顯眼。下一次大型賽事（美洲盃、歐洲盃）活動至少提前2週上線，入口放在下注主流程或APP首頁，不要只靠活動頁Banner。這是這次追蹤數字間接指出的問題（Old Event吃掉99.8%流量，代表玩家的下注路徑根本沒有被世界盃活動攔截到）。

下次改用增量當目標，不要再用事件期間的絕對量。這次 1B 定的是賽事期間全平台絕對量，結果 35.1%。絕對量目標有個結構性缺陷：97.7% 的量本來就來自 Organic 跟 Affiliate 的既有大盤，MKT 做得再好或再差，達成率的分子絕大部分都不是 MKT 能動的。下次用「賽事期間較平時基準提升 X%」這種增量型目標，分子才是活動真正能影響的部分，達成與否也才量得出 MKT 的表現。這次的基準數字已經量出來了：基期 39 天 $340M，世界盃帶動全平台 +3.1%、Sportsbook +14.8%，下次的目標就從這裡起算。

資料來源與方法論說明

  • 口徑：SettledTime（GMT+8）、TO=SUM(ValidBet)、GGR=−SUM(WinlostAmount)、Margin=GGR÷Valid TO、SettlementType IN (1,3)、Normal 會員且排除測試帳號。範圍為 OT（12bet）單一平台。
  • 來源：核心加總數字（TO/GGR/Margin/Betcount）以 Admin 後台為權威值；維度細節（Event/Source/Group/Device/Currency/Provider/每日趨勢）以 Looker 2026-07-21 匯出的完整 39 天資料為來源。兩套口徑本期差距 2.26%~2.61%，一律以 Admin 為準。
  • 基準期（05-04~06-11）：由底層注單表以同一套口徑重建，經兩條獨立路徑對數一致、另由獨立 agent 單日抽樣驗證。匯率採 39 天平均（抽樣驗證折算差異 0.13%），且與賽事期間可能存在季節性差異；對照表中 ±1% 以內的變化（GGR）應視為持平。
  • 已知限制：賽事最後 4 天的核心數字為 DB 重建值（與 Admin 有 0.6%~1.0% 已知落差）；Event/Source/Group 為三個獨立維度，不可跨維度直接比較。完整方法論細節見《2026世界盃・OT 1B成果總覽（FINAL版）》報告。
  • 全數字經 fresh agent 找碴式獨立覆核。
